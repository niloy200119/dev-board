Question-1: What are the different ways to select an element in the DOM?

There are several ways to select elements in the DOM, each serving different use cases. The simplest method is document.getElementById(), which retrieves a single element by its unique ID. For selecting multiple elements with the same class, document.getElementsByClassName() returns a live HTMLCollection, while document.getElementsByTagName() does the same for elements with a specified tag name (e.g., <div> or <p>). For more advanced and flexible selections, document.querySelector() returns the first element matching a CSS selector (e.g., ".class", "#id", or "div > p"), whereas document.querySelectorAll() returns a static NodeList of all matching elements, allowing for complex selections similar to CSS styling. These methods provide versatile ways to interact with DOM elements dynamically.

Question-2: What is the difference between innerHTML, innerText and textContent?

Here's a concise breakdown of the differences between innerHTML, innerText and textContent:

  1. innerHTML: Gets or changes everything inside an element, including words and tags (like <b>bold</b>). If you want to add new buttons or links, use this!
  2. innerText: Only shows the words you can see on the screen. If something is hidden (like with display: none), it won't show up.
  3. textContent: Gives you all the words inside an element, even if they're hidden. It doesn't care about tags (like <b>bold</b>)—just plain text!

Question-3: What is event delegation in the DOM?

Event delegation uses a single parent listener to handle events for multiple children by leveraging event bubbling. Instead of attaching listeners to each child, you check event.target to identify the source element. This improves performance and automatically works with dynamically added elements. It's ideal for lists, tables, and interactive UIs.

Question-4: What is event bubbling in the DOM?

Event bubbling in the DOM means that when an event (like a click) happens on an element, it first triggers on that element, then "bubbles up" through its parent elements all the way to the document root. This default behavior allows you to handle events efficiently—like listening for clicks on a parent element instead of each child—making it perfect for dynamic content and performance optimization. You can stop bubbling with event.stopPropagation() if needed, but most of the time, this upward flow simplifies event handling through delegation.

Question-5: How do you create, add and remove elements using JavaScript?

In JavaScript, we can create an element with document.createElement('div'), add it to the DOM using methods like parentElement.appendChild(newElement) or insertBefore(), and remove elements with parent.removeChild(child) or the modern child.remove().
